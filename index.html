❓ 常见面试问题与答案

VPC 网络架构怎样的？数据包怎么走？
详见《VPC 网络架构深度解析》。虚机流量经过虚拟交换机、VPC 路由器、安全组、NAT 网关，完整 3 层分层架构。虚机内网通过 OVS/Linux Bridge 转发，跨子网经过虚拟路由器（VyOS）。
Keepalived 漂移后业务不通，怎么排查？
详见《Keepalived+VRRP 工作原理》。从 5 个层面排查：①VIP 位置是否正确转移 ②虚机 ARP 表是否更新 ③conntrack 连接是否正常 ④防火墙规则是否允许 ⑤物理网络是否故障。
iptables 五链四表讲一下，为什么用 ipset？
详见《iptables 五链四表与 ipset 优化》。五链：PREROUTING、INPUT、FORWARD、OUTPUT、POSTROUTING。四表：raw、nat、mangle、filter。传统方式 10000 条规则耗时 10ms，ipset 方式耗时 0.1ms，性能提升 100 倍！
安全组更新如何做到不闪断、无感知？
使用 ipset 原子交换。创建新集合 ipset → 批量添加规则 → 原子 swap 交换 → 删除旧集合。整个过程在纳秒级完成，虚机完全无法感知到规则变化，零中断。
脑裂现象是什么？怎么解决？
两个 Master 同时持有 VIP，数据混乱。原因：网络分割导致心跳中断。解决：①多路径脑裂检测（ICMP、TCP、UDP、本地接口）②优先级下降 ③故障通知脚本 ④VIP 自动禁用 ⑤告警通知管理员。
虚机 ping 网关通，但上不了外网，原因？
3 种原因：①出站安全组规则禁止（检查 FORWARD 链）②NAT/EIP 规则不存在（检查 nat 表 POSTROUTING）③rp_filter 启用导致返程包被丢（检查 /proc/sys/net/ipv4/conf/*/rp_filter）。详见排查指南。
conntrack 是什么？什么时候会丢连接？
连接跟踪表，记录 NAT 映射和连接状态。当表满（99%+）时，新连接被标记为 INVALID，建立失败。解决：①增加 net.nf_conntrack_max ②启用连接复用 ③关闭无用连接 ④监控告警。
云平台怎么高可用？有没有脑裂？怎么解决？
主备 Keepalived + VRRP。脑裂风险确实存在（网络分割场景）。解决方案：①仲裁脚本多路径检测 ②健康检查机制 ③故障通知脚本 ④优先级自动调整 ⑤完整的监控告警体系。
x86 迁移 ARM 最容易遇到什么问题？
①GCC 版本不同导致编译错误 ②指令集差异（AVX vs NEON）③依赖库缺失或版本不兼容 ④字节序问题（大小端）⑤浮点精度差异 ⑥原子操作差异。详见《ARM 国产化兼容实战》完整解决方案。
一套平台兼容 x86/ARM，构建、发布、部署怎么做？
矩阵构建（x86 + ARM 并行）→ 多架构 RPM 仓库（自动分类）→ 架构自动识别脚本 → 灰度发布（同步进行）→ 统一升级脚本。详见《统一构建+多架构发布 CI/CD》附完整流水线配置。
编译优化怎么从 2h 降到 1h？
三阶段：阶段 1 并发编译（省 30 分钟）- 将串行改为并发 → 阶段 2 包拆分（省 15 分钟）- 大包拆小 → 阶段 3 缓存优化（省 15 分钟）- ccache + 分层缓存。详见《编译构建优化》附完整 Python 脚本和性能数据。
等保二级做过哪些强制安全项？
①root 权限收敛和最小权限 ②三权分立（管理、执行、审计）③密码策略强化 ④SSH 加固 ⑤操作审计和日志落地 ⑥权限表管理 ⑦加密存储和传输 ⑧监控和告警。详见《等保二级改造与落地》完整落地方案。
